Methodology: New Visit Elasticity
Elasticity Model

Price elasticity of demand measures how acquisition rates respond to price changes.

Core Formula
% Change in Acquisitions = Elasticity Coefficient √ó % Change in Price

Example: If elasticity = -2.1 and price increases by 10%, acquisitions decrease by 21%

Elasticity Coefficients (Industry Benchmarks)
Tier Base Elasticity Source
Standard Pass -2.1 Most price-sensitive (value seekers)
Premium Pass -1.7 Moderately sensitive (quality conscious)
Adjustment Factors

Base elasticity is modified by:

  • Tenure: New users (0-3 months) are 1.3√ó more sensitive
  • Age Group: 18-24 demographic is 1.2√ó more sensitive
  • Acquisition Channel: Paid ads users are 1.4√ó more sensitive
  • Time Horizon: Long-term effects (12+ months) are 1.5√ó stronger
Scenario Simulation
  1. Calculate price change percentage: (New Price - Current Price) / Current Price
  2. Apply elasticity: Demand Change = Elasticity √ó Price Change%
  3. Project new acquisitions: New Acq = Baseline Acq √ó (1 + Demand Change)
  4. Calculate revenue impact: New Rev = New Acq √ó New Price √ó Retention Rate

Model Implementation python/acquisition_model.py

Algorithm: Poisson Generalized Linear Model

Execution: Runs in browser via Pyodide (Python + NumPy in WebAssembly)

Link Function: log(E[adds]) = Œ≤‚ÇÄ + Œ≤‚ÇÅ√óprice + Œ≤‚ÇÇ√ódiscount + Œ≤‚ÇÉ√óis_promo + Œ≤‚ÇÑ√ósegment

Pre-fitted Coefficients:

  • intercept = 6.5 ‚Üí baseline ~665 adds/month
  • price = -0.25 ‚Üí price sensitivity

Why Poisson? Visitor counts are whole numbers (0, 1, 2...). This statistical method naturally fits count data and auto-adjusts as volumes change.

Important: Model coefficients are industry benchmarks, not trained on this synthetic dataset. Real-world deployment requires retraining on actual data.
Methodology: Loyalty Elasticity
Loyalty Model

Churn elasticity models how price increases drive visitor attrition over time.

Churn Formula
Additional Churn% = Loyalty Elasticity √ó Price Change% √ó Time Factor

Example: 10% price increase with -1.8 elasticity ‚Üí +18% return rate (baseline 3% ‚Üí 3.54%)

Time-Lagged Effects

Churn doesn't happen instantly. We model temporal dynamics:

Period Effect Multiplier Explanation
0-1 month 0.3√ó Immediate reaction (angry users)
2-3 months 1.0√ó Peak impact period
4-6 months 0.7√ó Tapering off (remaining users adjusted)
7-12 months 0.4√ó Long-tail effect
Cohort-Specific Churn

Different visitor segments have different retention sensitivities:

  • New visitors (0-6 months): 2.5√ó baseline loyalty elasticity
  • Mid-tenure (6-18 months): 1.0√ó baseline
  • Loyal (18+ months): 0.5√ó baseline (more forgiving)
  • Bundle visitors: 0.3√ó baseline (highest retention)
Retention Tactics

Model includes optional retention strategies:

  • Win-back campaigns: Can reduce churn by 15-25%
  • Grandfathering: Exempt existing users from price increase
  • Loyalty discounts: Offer 20% off for long-tenure users
ML Model: Logistic Regression (Python)
Model Implementation python/churn_model.py

Algorithm: Logistic Regression with time-lagged effects

Execution: Runs in browser via Pyodide (Python + NumPy)

Link Function: logit(churn) = Œ≤‚ÇÄ + Œ≤‚ÇÅ√óprice_change + Œ≤‚ÇÇ√ótime_interaction

Pre-fitted Coefficients:

  • price_x_0_4wks = 0.006 ‚Üí immediate effect multiplier

Why Logistic? Binary outcome (churn yes/no) requires logit link. Time-lagged interactions capture delayed churn response to price changes.

Data Source: Churn patterns calibrated from weekly_aggregated.csv historical return rates and industry research.
Methodology: Pass Type Migration
Migration Model

Models visitor movement between pricing tiers when relative prices change.

Migration Formula
Migration Rate = Base Rate √ó Price Gap Factor √ó Elasticity

Where Price Gap = (Higher Tier Price - Lower Tier Price) / Lower Tier Price

Migration Patterns
Downgrade Triggers (Premium Pass ‚Üí Standard Pass)
  • Base downgrade rate: 2-5% monthly
  • Increases by 1.5% for every $1 price gap increase
  • Peak downgrade occurs 2-3 months after price change
Upgrade Triggers (Standard Pass ‚Üí Premium Pass)
  • Base upgrade rate: 3-7% monthly
  • Decreases by 0.8% for every $1 price gap increase
  • Motivated by ad fatigue, premium content
Simulation Logic
  1. Calculate new price gaps between all tier pairs
  2. Estimate migration probabilities using logistic regression
  3. Apply cohort-specific modifiers (tenure, engagement)
  4. Simulate month-by-month transitions using Markov chain
  5. Calculate net revenue impact accounting for mix shift
Elasticity Cross-Effects

Migration is influenced by relative elasticities:

Cross-Elasticity = -0.5 √ó (Tier A Elasticity - Tier B Elasticity)

Higher elasticity differential = more migration

ML Model: Multinomial Logit (Python)
Model Implementation python/migration_model.py

Algorithm: Multinomial Logistic Regression (Softmax)

Execution: Runs in browser via Pyodide (Python + NumPy)

Outputs: Migration probability matrix across all tier pairs

Supported Tier Structures:

  • 2-Tier: Standard Pass ‚Üî Premium Pass
  • 3-Tier Bundle: Standard Pass, Premium Pass, Bundle (Service + Premium)
  • 3-Tier Basic: Basic, Standard Pass, Premium Pass

Sample Coefficients (3-Tier Bundle):

  • premium_pass_to_bundle intercept = -0.5 ‚Üí moderate upgrade appeal
  • value_savings_pct = 0.03 ‚Üí 21% savings ‚Üí +0.63 utility
  • has_content_need = 0.8 ‚Üí Premium content drives 55-70% conversion
  • tenure_months = 0.012 ‚Üí loyal visitors more likely to upgrade

Why Multinomial Logit? Multiple discrete choices (stay, upgrade, downgrade, cancel). Softmax ensures probabilities sum to 1. Handles asymmetric cross-effects between tiers.

Calibration: Migration rates calibrated from historical visitor flow data in visitors.csv (tier_previous field).
Methodology: Visitor Cohorts
Segmentation Framework

3-axis behavioral segmentation framework creating 375 unique visitor segments (5 visit frequencies √ó 5 party types √ó 5 price sensitivities √ó 3 pass tiers).

Three Segmentation Axes
Axis 1: Visit Frequency (5 segments)
  • One-Time Visitors: First-time or very rare visitors (tourists)
  • Occasional Visitors: Visit 2-3 times per year
  • Regular Visitors: Visit 4-8 times per year
  • Frequent Visitors: Visit 9-15 times per year
  • Season Pass Holders: 16+ visits per year, highest loyalty
Axis 2: Party Composition (5 segments)
  • Solo Visitors: Individual adults visiting alone
  • Couples: Two adults, typically without children
  • Small Families: 2-4 people, often with young children
  • Large Families: 5+ people, extended family groups
  • Groups: School trips, birthday parties, organized events
Axis 3: Price Sensitivity (5 segments)
  • Budget-Conscious: Highly price-sensitive, wait for deals
  • Value-Seekers: Want best value for money
  • Standard Spenders: Moderate price sensitivity
  • Premium Buyers: Willing to pay more for convenience
  • Luxury Enthusiasts: Price-insensitive, prioritize experience quality
Segment Elasticity Calculation
Segment Elasticity = Base Tier Elasticity √ó Visit Frequency Factor √ó Party Size Factor √ó Price Sensitivity Factor

Example: Occasional visitors, large families, budget-conscious (Standard Pass):
-1.8 (base) √ó 1.3 (occasional) √ó 1.2 (large family) √ó 1.5 (budget) = -4.21 (highly price-elastic)
‚Üí A 10% price increase would reduce visits from this segment by 42%

Contrast: Season pass holders, couples, premium buyers (VIP Pass):
-0.4 (base) √ó 0.6 (season pass) √ó 0.9 (couple) √ó 0.7 (premium) = -0.15 (highly inelastic)
‚Üí A 10% price increase would only reduce visits by 1.5%

Compares price sensitivity across visitor segments to identify optimal pricing strategies.

Analysis Methods
1. Elasticity Heatmap

Visual matrix showing elasticity coefficients across segments.

  • Color scale: Green (inelastic -0.5 to -1.5) ‚Üí Red (highly elastic -3.0+)
  • Sorting: By elasticity magnitude or revenue contribution
  • Filtering: By tier, acquisition channel, engagement level
2. Scatter Plot Analysis

2D visualization: Elasticity (X-axis) vs Revenue Impact (Y-axis)

  • Quadrant I: High elasticity, high revenue ‚Üí Risky to increase prices
  • Quadrant II: Low elasticity, high revenue ‚Üí Safe to increase prices
  • Quadrant III: Low elasticity, low revenue ‚Üí Low priority
  • Quadrant IV: High elasticity, low revenue ‚Üí Discount opportunity
3. 3D Radial Chart

Multi-dimensional view plotting segments across three axes:

  • Radius: Elasticity magnitude
  • Angle: Revenue contribution
  • Color: Churn risk level
Filtering & Aggregation

Analysis supports multiple filter combinations:

  • By Tier: Compare within Standard Pass or ad-free tiers
  • By Tenure: New (0-6mo), mid-tenure (6-18mo), loyal (18+mo)
  • By Engagement: Heavy users (>20hr/month), light users (<5hr /month)
  • By Risk: High retention risk (>10% monthly), stable (<3% monthly)

Key Insights

  • Segments with elasticity > -1.0 are inelastic (price increase opportunity)
  • Focus price optimization on high-revenue, low-elasticity segments
  • Consider targeted discounts for high-elasticity, low-engagement segments
